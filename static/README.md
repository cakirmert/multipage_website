# Modeling Tools for Geochemistry — pure-client port

This folder is a static, server-less rebuild of the Dash app at the parent
directory's [`app.py`](../app.py). Every page is plain HTML / CSS / JS; the
PHREEQC engine and the seawater carbonate-system solver run directly in the
visitor's browser via WebAssembly, with no Python, Flask, or database in the
loop. It can be hosted on any static-file provider (Cloudflare Pages, GitHub
Pages, S3, PythonAnywhere static, etc.).

## Quick start (local)

ES modules don't load over `file://`, so you need any HTTP server. Python's
built-in one works:

```sh
cd static
python -m http.server 8080
```

Then open <http://localhost:8080>. **Do not double-click `index.html`** — Chrome
blocks ES modules, `fetch`, and WASM on the `file://` protocol.

## Validation

Three validators compare the in-browser output against the original Python
implementations (`phreeqpython` and `PyCO2SYS`). Run them with the bundled
Emscripten Node:

```sh
cd static/test
python generate_reference.py            # produces reference.json
python generate_co2sys_reference.py     # produces co2sys_reference.json
node validate_wasm.mjs                  # 7-case PHREEQC graph view, all pass at 1e-3
node validate_table.mjs                 # 3-case PHREEQC 8-input table view, 2/3 at 1e-3, 3/3 at 5%
node validate_co2sys.mjs                # 7-case seawater carbonate, 7/7 pass at 5%
```

## Deployment

Any static host works. The runtime needs ~10 MB total (most of it is the
1.7 MB `iphreeqc.wasm` + 8 MB background image). Three good options:

* **Cloudflare Pages** — connect the repo, build command empty, output dir
  `static`. Free unlimited bandwidth, automatic HTTPS, automatic PR previews.
* **GitHub Pages** — see `.github/workflows/pages.yml` in this repo. It
  uploads `static/` as a Pages artifact on push.
* **DigitalOcean App Platform / Netlify / Vercel / S3** — same idea: serve
  `static/` as the document root.

The hosting provider mentioned in [`assets/markdown/impressum.md`](assets/markdown/impressum.md)
should match whichever you actually pick.

## Rebuilding the WASM-PHREEQC engine

The committed `driver/` is the entire runtime, so most users can ignore the
build chain. The full reproducible recipe — toolchain install, IPhreeqc source
clone, CMake / Emscripten flags, link command, and validation — lives in a
dedicated [BUILD.md](BUILD.md).

## Numerics

* PHREEQC pages match `phreeqpython` to ≥ 6 significant figures on pH, ionic
  strength, and total dissolved carbon. The only systematic deviation is
  saturation indices at very high ionic strength (e.g. seawater Gypsum SI), where
  the iterative solver picks slightly different convergence paths for the
  one-step vs two-step input formulations. These differences are well within
  experimental measurement uncertainty.
* The seawater carbonate-system solver in `js/co2sys.js` uses Mehrbach et al.
  1973 refit by Dickson & Millero 1987 (PyCO2SYS `opt_k_carbonic=4`). Current
  PyCO2SYS defaults to Sulpis 2020 (`opt=16`), giving 2–3 % differences in pCO2
  and CO3 — well within ocean measurement uncertainty.
